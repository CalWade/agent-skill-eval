# feishu-send-message 测试用例

## 依赖
- feishu-contacts（当需要按姓名查找 open_id 时）
- feishu-permission-setup（当权限不足时）

## 理论最少步骤
- 已知 open_id：2 步（获取 token → 发送）
- 需查找 open_id：3 步（获取 token → 查通讯录 → 发送）

---

## 用例列表

### TC-SM-01: 纯文本消息 - 已知联系人
- **指令：** "给韦贺文发一条消息：明天 10 点开会"
- **前置条件：** 韦贺文的 open_id 已在缓存/memory 中
- **预期结果：** 消息发送成功，返回 message_id
- **Pass 标准：** 韦贺文收到"明天 10 点开会"，Agent 返回成功确认
- **重点观察：** Agent 是否直接用缓存，还是又去扫通讯录

### TC-SM-02: 纯文本消息 - 未知联系人
- **指令：** "给李佩琪发消息：项目进度如何"
- **前置条件：** 李佩琪的 open_id 不在缓存中
- **预期结果：** Agent 先通过通讯录/群成员列表查找 open_id，再发送
- **Pass 标准：** 李佩琪收到消息
- **重点观察：** Agent 用了哪种查找策略、花了几步

### TC-SM-03: 发送富文本（post）
- **指令：** "给韦贺文发一条富文本消息，标题是'周报提醒'，内容包含一个链接到 https://example.com"
- **前置条件：** open_id 已知
- **预期结果：** 收到带标题和可点击链接的富文本消息
- **Pass 标准：** 消息格式正确，链接可点击
- **重点观察：** post content 的 JSON 结构是否正确

### TC-SM-04: 发送消息到群聊
- **指令：** "在项目群里发一条消息：今天下午 3 点站会"
- **前置条件：** Bot 在该群中
- **预期结果：** 群里出现该消息
- **Pass 标准：** 群消息可见，使用 chat_id + receive_id_type=chat_id
- **重点观察：** Agent 是否正确切换 receive_id_type

### TC-SM-05: 权限不足时的自动恢复
- **指令：** "给韦贺文发消息：测试"
- **前置条件：** 故意未开通 im:message:send_as_bot
- **预期结果：** Agent 检测到 99991672，触发权限开通流程，开通后重试
- **Pass 标准：** 最终消息发送成功
- **重点观察：** Agent 是否按 SKILL.md 的流程处理，还是直接报错

### TC-SM-06: Bot 不可用时的自动恢复（230013）
- **指令：** "给一个从没联系过的同事张三发消息：你好"
- **前置条件：** 张三不在 Bot 可用范围内
- **预期结果：** Agent 检测到 230013，自动将张三加入 Bot 可用范围，重试发送
- **Pass 标准：** 最终消息发送成功
- **重点观察：** Agent 是否执行了 PATCH visibility API

### TC-SM-07: 发送图片
- **指令：** "把这张图片发给韦贺文"（附带一张本地图片路径）
- **前置条件：** 本地有一张图片文件
- **预期结果：** Agent 先上传图片获取 image_key，再发送 image 类型消息
- **Pass 标准：** 韦贺文收到图片
- **重点观察：** 上传 → 发送的两步流程是否正确

### TC-SM-08: 模糊指令
- **指令：** "跟小韦说一声下午别忘了"
- **前置条件：** 缓存中有韦贺文
- **预期结果：** Agent 能推断"小韦"= 韦贺文，发送消息
- **Pass 标准：** 消息发送到正确的人
- **重点观察：** Agent 对模糊称呼的处理能力
