# 评估维度定义

## 五个核心指标

### D1. 首次成功率（First-Try Success Rate）

**定义：** 用户发出一句话指令后，Agent 不经任何人工干预、不需用户补充信息，一次跑通完整流程的比例。

**计算：** `成功次数 / 总测试次数 × 100%`

**判定标准：**
- ✅ Pass：Agent 完成了用户意图，产出正确结果（消息已发送 / 文档已创建 / 数据已写入等）
- ❌ Fail：中途报错停止、结果错误、需要用户额外提供信息、Agent 卡死无响应

**为什么重要：** 这是小白用户体验的第一指标。用户不会容忍"再试一次"。

---

### D2. 步骤效率（Step Efficiency）

**定义：** Agent 完成任务实际执行的工具调用步骤数 vs 理论最少步骤数。

**计算：** `理论最少步骤 / 实际步骤 × 100%`

**理论最少步骤的确定方法：**
- 根据 SKILL.md 中的核心流程，数一下必需的 API 调用数
- 例如 send-message：① 获取 token → ② 发送消息 = 2 步（已知 open_id 时）
- 例如 send-message：① 获取 token → ② 查通讯录 → ③ 发送消息 = 3 步（需找人时）

**为什么重要：** 冗余步骤 = 浪费 token = 更容易卡死 = 响应更慢。

---

### D3. 错误恢复率（Error Recovery Rate）

**定义：** 当 API 返回已知错误码（99991672 / 230013 / 99991663 等）时，Agent 能否按 SKILL.md 的指引自动处理并最终完成任务。

**计算：** `自动恢复次数 / 触发错误总次数 × 100%`

**测试方法：** 故意制造错误场景：
- 不开通权限，看 Agent 是否触发 permission-setup
- 用过期 token，看 Agent 是否自动刷新
- 用无效 open_id，看 Agent 是否降级到其他查找方式

**为什么重要：** 零基础用户无法自己处理 API 报错。Agent 的自愈能力决定了用户是否需要找技术人员。

---

### D4. 输出质量（Output Quality）

**定义：** Agent 完成任务后，返回给用户的信息是否完整、准确、可操作。

**评分标准（1-5 分）：**

| 分数 | 标准 |
|------|------|
| 5 | 结果正确 + 包含所有关键信息（链接/ID/状态） + 格式清晰 |
| 4 | 结果正确 + 有关键信息，但格式一般 |
| 3 | 结果正确，但缺少部分关键信息（如没给文档链接） |
| 2 | 结果部分正确（如文档创建了但内容写入不全） |
| 1 | 结果错误或无法使用 |

**为什么重要：** 任务完成 ≠ 用户满意。链接打不开、信息不全、格式乱，都会让用户困惑。

---

### D5. 上下文消耗（Context Consumption）

**定义：** Agent 完成任务过程中消耗的总 token 数（输入 + 输出），或等效衡量指标（Agent 回复轮次数）。

**衡量方式：**
- 优先：直接从 API 响应中读取 token usage（如果平台提供）
- 次选：数 Agent 的回复轮次（每轮大约对应一定量的 token）
- 兜底：记录 Agent 从开始到完成的总字符数

**阈值参考：**
- 🟢 优秀：≤ 3 轮回复
- 🟡 合格：4-6 轮回复
- 🔴 需优化：> 6 轮回复 或 出现卡死/超时

**为什么重要：** 这是 PDF 对话中"卡 10 分钟"问题的量化指标。上下文越长，卡死概率越高，成本越高。

---

## 优先级

| 优先级 | 指标 | 原因 |
|--------|------|------|
| P0 | D1 首次成功率 | 最核心，直接决定用户能不能用 |
| P0 | D3 错误恢复率 | 零基础用户的生命线 |
| P1 | D5 上下文消耗 | 直接影响能否完成 + 成本 |
| P1 | D4 输出质量 | 影响用户信任度 |
| P2 | D2 步骤效率 | 优化项，不影响功能 |

---

## 失败原因分类（Failure Taxonomy）

每次测试失败时，记录属于以下哪一类：

| 代码 | 分类 | 示例 |
|------|------|------|
| F1 | **Skill 文档缺陷** | SKILL.md 流程有遗漏、API 参数写错 |
| F2 | **Agent 理解偏差** | Agent 没按 SKILL.md 执行，自己编了一套 |
| F3 | **权限/配置问题** | 权限未开通、App 配置不对 |
| F4 | **飞书 API 限制** | 接口本身有bug、返回不符合文档 |
| F5 | **上下文溢出** | Token 超限、Agent 卡死 |
| F6 | **用户指令歧义** | 指令太模糊，Agent 无法判断意图 |
| F7 | **跨 Skill 依赖失败** | 需要联动其他 Skill（如 contacts）但失败 |
